# LinkedIn Content Calendar for dbt-Workbench

## Content Strategy

### Target Audience
- **Primary:** Data Engineers, Analytics Engineers
- **Secondary:** Data Analysts, Data Team Leads, Engineering Managers

### Content Pillars
1. **Product Features (35%)** - Feature highlights, tutorials, tips & tricks
2. **Community Building (25%)** - Contributor stories, open source philosophy
3. **Use Cases (20%)** - Success stories, integrations, workflows
4. **Industry Thought Leadership (15%)** - Data engineering trends, best practices
5. **Behind-the-Scenes (5%)** - Team updates, development insights

### Posting Frequency
- **Tuesday:** Feature highlight or tutorial
- **Thursday:** Community or use case
- **Friday:** Light content (meme, quote, open source motivation)
- **Monthly:** Release notes (last Tuesday of month)

### Hashtag Strategy
**Always include 3-5:**
- `#dbt #DataEngineering #OpenSource`
- Add 1-2 contextual: `#DataLineage`, `#AnalyticsEngineering`, `#SQL`, `#DataPipeline`

### 2. Tutorial (Thursday)
**Format:** Carousel or long-form post
**Structure:**
- Problem statement
- Step-by-step solution (3-5 steps)
- Code snippet/command
- Outcome/result
- CTA

### 3. Community Post (Fri/Sat)
**Format:** Quote, meme, or short story
**Tone:** Light, engaging, relatable

### 4. Poll (Mon)
**Format:** LinkedIn native poll
**Topics:** Pain points, preferences, tool comparisons
**Follow-up:** Comment with results/insights

### 5. Carousel (Weekly 1-2x)
**Structure:**
- Slide 1: Title
- Slides 2-8: Content (one point per slide)
- Last slide: CTA + QR code

- Within 24 hours
- Ask follow-up questions
- Tag team members when appropriate

### LinkedIn Groups
- Join: dbt, Data Engineering, Open Source groups
- Share valuable content (not just self-promo)
- Participate in discussions

## Content Approval Process

1. **Draft creation** (content creator)
2. **Review** (admin, 24h turnaround)
3. **Schedule** (LinkedIn scheduling tool)
4. **Post** (scheduled or live)
5. **Monitor** (first 24h engagement)
6. **Report** (weekly metrics)

## Tools & Resources

### Content Creation
- **Screenshots:** Clean UI captures, annotate with arrows/text
- **Carousels:** Canva, Figma, or PowerPoint
- **GIFs:** Loom or ScreenToGif
- **Images:** Unsplash, Pexels (open-source friendly)

### Scheduling
- LinkedIn native scheduler (up to 3 months)
- Hootsuite/Buffer (for cross-platform)

### Analytics
- LinkedIn Page Analytics (native)
- Weekly export to Google Sheets
- Monthly report deck

## Crisis Management

If post receives negative feedback:
1. Monitor for 24h
2. Respond professionally to constructive criticism
3. Ignore/troll filter non-constructive comments
4. Learn and adjust future content
5. Document lessons learned
